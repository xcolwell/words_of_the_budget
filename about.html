whatwepayfor.org provides access to digital form of the federal budget. In the data, each year's budget is presented as budget spending accounts, where each account has one broad function, and one more specific sub-function. However, the name of the account, not the function or sub-function, seems to give the most understanding into the type of social benefit the account creates. From the view of a single word, one can see the benefit of the budget most succinctly. the words of the budget explores the US federal spending broken down by word.

Submitted as part of the March 2011 Data Visualization Challenge.

Brien Colwell, Life On the Digitally Enhanced Streets (LODES)

Data were pulled from the web API using "getBudgetAccount", using a Mathematica script. Some budget accounts have negative spending. I'm not sure the meaning of this, but I interpreted it as not spending. Some budget accounts have redundant names. I assume the government just stops trying to be creative for some of these.

After importing from the API, the Mathematica script writes a set of JavaScript files that load into the browser to supply the data to the visualization. The visualization is written in JavaScript using HTML5 canvas and audio extension.

Voices for meet the words of the budget were created with festival using the Nitech HTS voices.

Word geometry data was created with a custom program to enable a "word mesh" in the browser.

The complete source code, scripts, and whatnot for the project is available under a Creative Commons Attribution license.
